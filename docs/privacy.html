Privacy Policy

What Keking reads, what it stores, and what happens to your data.

Keking plays music, speaks text, and fixes social media links. It needs basic data to do this.

1. Data We Read

When you invite Keking, the bot reads:

  • Server IDs and names
  • Your slash commands
  • Message text, but only to fix links or answer mentions
  • Voice channel status so it can join and play audio

2. How We Use It

We use this data to make the bot work. Specifically:

  • Answering slash commands
  • Playing your music and TTS requests
  • Detecting Twitter or Instagram links to reply with working previews

3. Data We Store

If you enable the database, Keking stores basic server info:

  • Guild ID and name
  • Command hash (to know if commands need updating)
  • Join date

Keking stores no message text, no user profiles, and no chat history.

4. Message Content

Keking scans messages in real time to fix bad social media links. We do not save this text. We do not use it to build profiles.

5. AI Training

We do not train AI models on your messages or data.

6. Data Sharing

We sell nothing. We share nothing. Keking only sends queries to external APIs (like Genius or YouTube) when you ask it to find lyrics or songs.

7. Deletion

Kick the bot, and it stops reading data. If you have the database running, it deletes your server's record.

8. Contact

Create an issue on GitHub if you need help with your data.
